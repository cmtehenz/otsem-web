Aqui está como usar no seu frontend:

Exemplo de Uso no Frontend
1. Solicitar KYC (botão "Verificar Identidade")
// Função para iniciar verificação KYC
async function iniciarVerificacaoKYC(customerId: string, token: string) {
  const response = await fetch(`${API_URL}/customers/${customerId}/kyc/request`, {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${token}`,
      'Content-Type': 'application/json',
    },
  });
  const data = await response.json();
  
  // Redireciona o usuário para a página de verificação do Didit
  if (data.verificationUrl) {
    window.location.href = data.verificationUrl;
  }
  
  return data;
}

2. Consultar Status do KYC
// Função para consultar status
async function consultarStatusKYC(customerId: string, token: string) {
  const response = await fetch(`${API_URL}/customers/${customerId}/kyc/status`, {
    method: 'GET',
    headers: {
      'Authorization': `Bearer ${token}`,
    },
  });
  const data = await response.json();
  // data.accountStatus = 'not_requested' | 'requested' | 'in_review' | 'approved' | 'rejected'
  // data.diditDecision = decisão detalhada do Didit (se disponível)
  
  return data;
}

3. Configurar Webhook no Didit
No painel do Didit, configure a URL do webhook:

https://seu-dominio.com/didit/webhooks/verification

O Didit enviará automaticamente o resultado da verificação para essa URL, e a API atualizará o status do customer.

Resposta do endpoint /kyc/request:

{
  "id": "customer_id",
  "accountStatus": "requested",
  "verificationUrl": "https://verification.didit.me/session/abc123",
  "sessionId": "abc123",
  "requestedAt": "2025-12-13T22:45:00.000Z"
}

Resposta do endpoint /kyc/status:

{
  "id": "customer_id",
  "accountStatus": "approved",
  "diditSessionId": "abc123",
  "diditVerificationUrl": "https://verification.didit.me/session/abc123",
  "diditDecision": {
    "status": "completed",
